# Phase 6: Government Policy Risk Engine (New Subsystem)

## Question 1: Where do we gather this data?
**Answer:** We already have it! The `all_schemes_master.csv` file we scraped from the government portal contains exactly 4,580 real-world schemes. The AI algorithms below will mathematically scan the `Description`, `Category`, and `Tags` of these 4,580 schemes to calculate their risk levels.

## Question 2: The Algorithms
To keep this completely isolated from the citizen recommendation engine, we will build a `government_risk_analyzer.py` script. This script will use Python's NLP text-processing capabilities to run the following 5 algorithms on every scheme:

### Algorithm 1: Accessibility / Exclusion Risk
Calculates how difficult it is for a poor or illiterate person to apply.
- **Logic:**
  1. Count the number of document requirements in the text (e.g., words like "certificate", "card", "proof", "affidavit"). Let this be `Doc_Count`.
  2. Check if the scheme forces digital literacy (e.g., words like "online portal", "upload", "website"). Let this be `Online_Penalty = 3`.
- **Formula:** `Risk Score = (Doc_Count * 0.5) + Online_Penalty` (Capped at 10)

### Algorithm 2: Bureaucratic Friction (Red Tape)
Calculates the risk of corruption and massive delays.
- **Logic:**
  1. Count the number of authoritative bodies required for approval (e.g., "department", "committee", "officer", "panchayat", "collector"). Let this be `Dept_Count`.
  2. Look for bottleneck keywords (e.g., "inspection", "verification", "audit"). Let this be `Delay_Penalty = 2`.
- **Formula:** `Risk Score = (Dept_Count * 1.5) + Delay_Penalty` (Capped at 10)

### Algorithm 3: Market Distortion / Dependency Risk
Calculates if a scheme builds long-term dependency (handouts) rather than empowering citizens.
- **Logic:**
  1. Count "Handout" keywords: "free", "subsidy", "financial assistance", "cash".
  2. Count "Empowerment" keywords: "training", "skill", "education", "loan", "entrepreneur".
- **Formula:** 
  - If `Handouts` > `Empowerment` -> `Risk Score = 7`
  - If `Handouts` heavily outweigh `Empowerment` and includes "monthly" -> `Risk Score = 9`
  - If `Empowerment` >= `Handouts` -> `Risk Score = 2` (Safe)

### Algorithm 4: Ecological / Environmental Risk
Calculates if agricultural/industrial schemes deplete natural resources.
- **Logic:**
  1. Target specific categories (Agriculture, Industry, Infrastructure).
  2. Scan for high-impact keywords: "fertilizer", "borewell", "diesel", "tractor", "pesticide", "groundwater". Let this be `Eco_Threats`.
- **Formula:** `Risk Score = Eco_Threats * 2.5` (Capped at 10)

### Algorithm 5: Social Friction Risk
Calculates if a scheme is so hyper-targeted that it excludes the majority, potentially causing demographic tension.
- **Logic:**
  1. Count the number of demographic filters applied (e.g., "caste", "tribe", "religion", "minority", "urban", "rural"). Let this be `Filter_Count`.
- **Formula:** `Risk Score = Filter_Count * 2` (Capped at 10)

### Final Output
- **Composite Risk Score:** `Average(Alg1, Alg2, Alg3, Alg4, Alg5)`
- The script will save these mathematical scores into a new database table `government_risk_analysis`.
- We will build a "Government Dashboard" UI that sorts the 4,580 schemes to show the Government exactly which ones are the most "dangerous" and *why*.
